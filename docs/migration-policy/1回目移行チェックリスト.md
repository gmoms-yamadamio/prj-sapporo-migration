# 1回目データ移行 作業チェックリスト

> 対象手順書: [1回目移行手順書.md](./1回目移行手順書.md)
> 目的: 移行結果に**過不足・不整合がないこと**を、各フロー・各作業実施時にチェック観点に基づいて確認する
> 記入要領:
> - 各チェック項目について、実施日・実施者・結果を記入し、判定列に `OK` / `NG` を記載する
> - `NG` の場合は備考に原因・対応方針・再確認結果を記載する
> - 「期待値・確認方法」列の数式（例: `1.3 = 1.1 - 1.2`）は、実施後に必ず突合すること
> - 「入力方法」列は本チェック表の記入を誰が/どう行うかを示す（詳細は次章「結果の自動入力について」）

## 0. 結果の自動入力について

チェック結果は、可能な限り ETL スクリプト実行時に自動入力する。手動で本表に転記する手間・記入漏れ・件数の転記ミスを減らすことが目的。

| 入力方法 | 内容 |
| --- | --- |
| **自動** | `scripts/run_all.py` 実行時に、ETLスクリプト（`generate_receipt_stats.py`/`generate_member.py`/`generate_order.py` 等）が計算済みの値を `output/reports/checklist_results.json` に記録する。値同士の突合（例: `3.3 = 3.1+3.2 = 1.3`）も自動判定する |
| **半自動（ログ解析）** | SB00092 の実行ログ（テキストファイル）を `python scripts/parse_sb00092_log.py <ログファイル>` に渡すと、既知のログパターン（`成功n件/エラーn件` 等）を解析して記録する。ログ実フォーマットが想定と異なる場合は解析できず、手動記録が必要 |
| **手動（CLI記録）** | CEC管理画面での確認結果等、スクリプトで直接取得できない値は `python scripts/record_manual_result.py <チェックID> <結果> --by <実施者> --judge OK` で記録する。自動項目と同じ集計ファイルに記録されるため、最終レポートに一元的に反映される |
| **手動（目視確認のみ）** | 画面表示確認・お客様レビュー等、値そのものに意味がない確認項目。上記CLIで結果（コメント）を記録するか、本表に直接手書きする |

自動・半自動・手動で記録された結果は、`python scripts/render_checklist_report.py` の実行により
`output/reports/checklist_report.md` に統合レポートとして出力される（`run_all.py` の最終ステップでも自動実行される）。

> **注意**: `checklist_report.md` は実施記録を補助する自動生成物であり、本チェックリスト（本ファイル）を置き換えるものではない。
> 最終的な承認・総合判定（0章）は、自動レポートの内容を確認した上で本ファイルに記入・保管すること。
