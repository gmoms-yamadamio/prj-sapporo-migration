# 会員 CSV 作成方針

> 全体方針: [作業方針設計書.md](../作業方針設計書.md)  
> 項目定義: [【サッポロビール様】CSVファイルIF設計書.xlsx](../【サッポロビール様】CSVファイルIF設計書.xlsx) シート「会員 CSV 項目」

## 概要

| 項目 | 内容 |
|---|---|
| 成果物 | 会員 CSV（**新規登録用・更新用の2ファイルに分けて出力**） |
| 配置先 | `output/processed/`（`member_import_create.csv` = パターンA新規`C`分、`member_import_update.csv` = パターンB更新`U`分） |
| 投入先 | CEC 管理画面インポート（新規用・更新用をそれぞれ個別にアップロード） |
| 作成方法 | スクリプト（赤星商店 CEC 会員との突合あり） |

> **【確定】** パターンA（新規`C`）とパターンB（更新`U`）は、1ファイルに操作区分列で混在させるのではなく、**別ファイルとして出力する**。

## 前提

シュパーク移行後の会員は、**クラウド EC で既に運用開始している「赤星商店」の会員情報と共用**する。  
旧サイト会員を弊社指定フォーマットへ変換する際、赤星商店の CEC 会員データとの突合が必須となる。

## 入力データ

| データ | 配置先 | 役割 |
|---|---|---|
| `User.csv` | `input/raw/` | 旧サイト会員マスタ |
| `UserAccount.csv` | `input/raw/` | 旧サイト会員アカウント |
| CEC 会員データ（赤星商店） | `input/cec/` | 既存登録済み会員との突合 |

## 生成ルール

旧サイト会員ごとに赤星商店 CEC 会員データと突合し、次の 2 パターンに振り分ける。

| パターン | 条件 | 操作区分 | 処理内容 |
|---|---|---|---|
| **A. 新規登録** | 赤星商店に未登録の会員 | `C`（新規） | 旧サイトデータをもとに会員 CSV の全項目を設定し、新規会員登録を行う |
| **B. 更新のみ** | 赤星商店に登録済みの会員 | `U`（更新） | **新規会員登録は行わない**。赤星商店の会員情報を正とし、以下の項目のみ更新する |

### パターン B で更新する項目（のみ）

| 項目 | 備考 |
|---|---|
| `[カスタム]メールマガジン配信（シュパーク）` | 旧サイトの配信設定を反映 |
| `[カスタム]旧会員ID` | 旧サイトの会員 ID（`User.Id`）を保持 |

### 補足

- パターン B では、CEC 会員 CSV の更新仕様に従い、**会員 ID（CEC 採番済み）を指定**して更新行を出力する
- 氏名・メールアドレス・住所等の基本属性は赤星商店側の既存データを維持し、上書きしない
- 突合キーはメールアドレスを第一候補とする（詳細はマッピング設計時に確定）

## 突合・振り分けフロー

```
旧サイト会員（User.csv + UserAccount.csv）
        │
        ├─ 赤星商店 CEC 会員に該当あり ──→ 操作区分 U（更新のみ）
        │                                  ・会員情報は赤星商店を正とする
        │                                  ・更新項目:
        │                                    - [カスタム]メールマガジン配信（シュパーク）
        │                                    - [カスタム]旧会員ID
        │
        └─ 赤星商店 CEC 会員に該当なし ──→ 操作区分 C（新規登録）
                                           ・旧サイトデータをもとに全項目を設定
```

## 突合レポート

突合結果は `output/reports/` にレポートとして出力する。

- 新規登録対象件数 / 更新のみ対象件数
- 突合不能（要手動確認）の会員リスト
- 旧サイト側で重複している会員の扱い

## 他成果物との関係

| 成果物 | 関係 |
|---|---|
| 会員アドレス帳 CSV | 会員メールアドレス欄の設定に本 CSV の突合結果を使用 |
| 会員ポイント CSV | 会員 ID 欄（暫定メールアドレス）の設定に突合結果を使用 |
| 注文履歴 CSV | 会員 ID（メールアドレス）欄は突合結果を経由せず、`User.csv` のメールアドレスを直接使用（2026-07-14 ロジック簡略化。[注文履歴CSV.md](./注文履歴CSV.md) 参照） |

## 項目マッピング（パターン A: 新規 `C`）

結合: `UserAccount.UserName` = `User.Id`（[join-keys.md](../../csv-interface/join-keys.md) 参照）

| 会員 CSV 項目 | 入力元 | 変換ルール |
| --- | --- | --- |
| 操作区分 | — | 固定 `C` |
| 名前 | `User.LastName` + `User.FirstName` | 半角スペース区切り結合 |
| 名前読み | `User.LastNameKana` + `User.FirstNameKana` | 半角スペース区切り。空の場合は `・` |
| メールアドレス | `User.Email` | そのまま |
| 生年月日 | `User.Birthday` | `YYYYMMDD` 形式（8 桁、区切りなし）に変換（**修正**。IF 設計書「会員 CSV 項目」シートの記載に基づく。従来 `YYYY/MM/DD` としていたのは誤り） |
| 性別 | `User.Sex` | `0`→`woman`, `1`→`man`, `2`/`9`→`other` |
| 電話番号1 | `User.PhoneNumber` | そのまま |
| 郵便番号 | `User.ZipCode` | そのまま |
| 都道府県 | `User.Prefecture` | 数値コード→都道府県名（1=北海道…47=沖縄県） |
| 市区町村 | `User.City` | そのまま |
| 番地 | `User.Street` | そのまま |
| 建物名等 | `User.Building` | そのまま |
| 登録元サイトID | — | 固定 `shupark`（IF 設計書「登録元サイトコード」の設定可能値は `shupark` または `eshop`） |
| [カスタム]メールマガジン配信（シュパーク） | `User.IsApproveMailDelivery` | `1`→`対象`, `0`→`対象外`（**修正**。IF 設計書の属性は「全角」で値は `対象`/`対象外`。従来の `true`/`false` は誤り） |
| [カスタム]旧会員ID | `User.Id` | そのまま（確定） |
| [カスタム]法人コード | — | **一律ブランク（確定）**。IF 設計書では新規登録時「必須」属性だが、シュパーク移行では法人 CSV・法人会員を対象外とする方針（[作業方針設計書.md](../作業方針設計書.md) §2.3）のため、値を設定せず空欄で出力する |

以下は IF 設計書「会員 CSV 項目」シートに定義があるが、旧サイトに対応データがないか、本移行では設定不要と判断した項目。

| 項目 | 属性 | 扱い |

| 会員登録日時 | 任意 | 未設定（CSV 登録時刻がそのまま設定される） |
| ニックネーム | 任意 | 未設定 |
| 会員グループコード | 任意 | 未設定（[「(参考)eshop会員グループコード」](../【サッポロビール様】CSVファイルIF設計書.xlsx)シートにシュパーク一般会員向けのコードが存在しない。`mg_007`＝eSHOP シュパーク管理者のみで対象外） |

| [カスタム]メールマガジン配信（eSHOP）/（ラガービール） | 任意 | 未設定（シュパーク以外のブランド向け項目） |

## 項目マッピング（パターン B: 更新 `U`）

赤星商店 CEC 会員（`input/cec/member_export.csv`）とメールアドレスで突合。

| 会員 CSV 項目 | 入力元 | 変換ルール |
| --- | --- | --- |
| 操作区分 | — | 固定 `U` |
| 会員ID | CEC エクスポート `会員ID` | 赤星商店側の CEC 会員 ID |
| [カスタム]メールマガジン配信（シュパーク） | `User.IsApproveMailDelivery` | `1`→`対象`, `0`→`対象外`（**修正**。従来の `true`/`false` は誤り） |
| [カスタム]旧会員ID | `User.Id` | そのまま（確定） |

その他の基本属性（氏名・メール等）は出力しない（赤星商店データを維持）。

## 突合ロジック（確定）

1. `UserAccount`（`IsAnonymous=0`）を起点に `User` を `UserName=Id` で結合
2. `User.DeletedAt` ありは除外（確定）
3. `User.Email`（lower 正規化）で `input/cec/member_export.csv` と突合
4. マッチ → パターン B（`U`）、非マッチ → パターン A（`C`）
5. 同一メールの重複は `UpdatedAt` 最新を採用、他はレポート出力
