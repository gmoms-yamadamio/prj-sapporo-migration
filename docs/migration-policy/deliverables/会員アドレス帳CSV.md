# 会員アドレス帳 CSV 作成方針

> 全体方針: [作業方針設計書.md](../作業方針設計書.md)  
> 項目定義: [【サッポロビール様】CSVファイルIF設計書.xlsx](../【サッポロビール様】CSVファイルIF設計書.xlsx) シート「会員アドレス帳 CSV 項目」

## 概要

| 項目 | 内容 |
|---|---|
| 成果物 | 会員アドレス帳 CSV |
| 配置先 | `output/processed/` |
| 投入先 | CEC 管理画面インポート |
| 作成方法 | スクリプト（一律追加登録） |

## 入力データ

| データ | 配置先 | 役割 |
|---|---|---|
| `User.csv` | `input/raw/` | 会員の基本情報（メールアドレス、氏名等） |
| `UserAccount.csv` | `input/raw/` | 会員アカウント（`UserNo` 等） |
| `UserAddress.csv` | `input/raw/` | 会員と住所の紐付け（`UserNo` ↔ `AddressId`） |
| `Address.csv` | `input/raw/` | 住所の詳細（氏名、郵便番号、都道府県等） |

## データの結合イメージ

```
UserAccount (UserNo)
    ├─→ User.csv          … 会員メールアドレス等の取得
    └─→ UserAddress.csv   … AddressId の取得
              └─→ Address.csv … 住所詳細の取得
```

## 生成ルール

| 条件 | 処理 |
|---|---|
| 全会員（共通） | 操作区分 `C`（新規追加）で会員アドレス帳 CSV を出力する |
| 赤星商店に登録済みの会員 | **登録済みアドレス帳との重複確認・差分更新は行わない** |
| 赤星商店に登録済みの会員 | **シュパークのアドレス帳を一律追加登録**する |

### 補足

- 赤星商店会員であっても、旧サイトの住所データは CEC 既存アドレス帳と突合せず、**すべて新規追加（`C`）** として投入する
- 会員 CSV で赤星商店突合済みの会員については、**会員メールアドレス** 欄に CEC 上の会員メールアドレス（赤星商店側）を設定する
- 会員 CSV で新規登録（`C`）となる会員については、旧サイト `User.csv` のメールアドレスを設定する
- アドレス ID は新規登録時は CEC 側で自動採番のため、CSV では出力しない（IF 設計書の新規登録仕様に従う）

## 会員 CSV との関係

| 会員 CSV のパターン | 会員アドレス帳 CSV の扱い |
|---|---|
| A. 新規登録（`C`） | 旧サイト住所を追加登録。会員メールアドレスは旧サイトの値 |
| B. 更新のみ（`U`） | 旧サイト住所を追加登録。会員メールアドレスは赤星商店 CEC 側の値 |

- 会員 CSV で基本属性を更新しない（パターン B）場合でも、アドレス帳は旧サイトデータをそのまま追加する
- 赤星商店側に同一住所が既に存在していても、重複排除は行わない（意図的な仕様）

## 処理順序

会員 CSV の突合・振り分けが完了した後に作成する。

## 項目マッピング

結合: `UserAccount.UserNo` → `UserAddress.UserNo` → `Address.addressId`（[join-keys.md](../../csv-interface/join-keys.md)）

| 会員アドレス帳 CSV 項目 | 必須 | 入力元 | 変換ルール |
| --- | --- | --- | --- |
| 操作区分 | 必須 | — | 固定 `C` |
| 会員メールアドレス | 必須 | 会員 CSV 突合結果 | パターン A: `User.Email` / パターン B: CEC メール |
| 分類 | 必須 | — | 固定 `0`（IF 設計書「(参考) 0：お届け先」より。**追加項目**。従来のマッピングに未記載だったため追加） |
| アドレス名 | 必須 | `UserAddress.AddressName` | そのまま（数値文字列の場合あり）。※ IF 設計書のサンプル値は `"お届け先住所（"+苗字+名+"）"` 形式の生成例だが、旧サイトの `AddressName` 列を優先して転記する方針とする（要確認、下記 TODO 参照） |
| お名前（苗字） | 必須 | `Address.recipientlastname` | そのまま（IF 設計書では苗字・名は別項目。結合しない） |
| お名前（名） | 必須 | `Address.recipientfirstname` | そのまま |
| お名前（フリガナ）（苗字） | 必須 | `Address.recipientlastnamekana` | そのまま |
| お名前（フリガナ）（名） | 必須 | `Address.recipientfirstnamekana` | そのまま |
| 郵便番号 | 必須 | `Address.zipCode` | そのまま |
| 都道府県 | 必須 | `Address.pref` | マスク値の場合は `User.Prefecture` から補完を検討 |
| 市区町村 | 必須 | `Address.city` | そのまま |
| 番地 | 必須 | `Address.street` | そのまま |
| 建物名等 | 任意 | `Address.building` | そのまま |
| 電話番号1 | 必須 | `Address.tel` | そのまま |

> 2026-07-09 修正: IF 設計書「会員アドレス帳 CSV 項目」シートでは「お名前（苗字）」「お名前（名）」は別項目として定義されており、事前に結合して 1 項目にするものではない。従来ドキュメントの「結合」という記載を削除し、各項目を個別に転記する方針に修正した。

## 出力ルール（確定）

| 条件 | 処理 |
| --- | --- |
| 1 会員に複数住所 | 住所ごとに 1 行（すべて `C`） |
| `UserAddress` 未登録会員 | **出力スキップ** |
| 削除済み会員 | **移行対象外**（`User.DeletedAt` あり） |
