# 商品系 Excel 作成方針

> 全体方針: [作業方針設計書.md](../作業方針設計書.md)  
> 項目定義: [【サッポロビール様】CSVファイルIF設計書.xlsx](../【サッポロビール様】CSVファイルIF設計書.xlsx)

## 概要

`Products.csv`（3 行ヘッダ）を Excel に取り込み、関数で商品マスタ・販売基本情報・販売条件の 3 シートを生成する。

## Excel 構成

| シート | 役割 |
| --- | --- |
| Products | `input/raw/Products.csv` の取込（行 4 以降がデータ） |
| 商品マスタ | 関数により弊社指定フォーマットへ変換 |
| 販売基本情報 | 同上 |
| 販売条件 | 同上 |
| マスタ・変換表 | サイト ID、管理グループコード等 |

## Products.csv 列定義（英語ヘッダ行）

| 列名 | 日本語 | 型 | 用途 |
| --- | --- | --- | --- |
| Id | 内部商品ID | Int32 | 旧サイト商品主キー。注文明細の productId と対応 |
| ExternalId1 | 外部ID1 | String | **SKUコード / 商品コード** として使用 |
| ExternalId2 | 外部ID2 | String | 予備 |
| Name | 管理名 | String | 商品名 |
| SalesStatus | 販売ステータス | Int32 | 販売可否 |
| SalesStart | 販売開始日時 | DateTime | |
| SalesEnd | 販売終了日時 | DateTime | |
| Stockout | 在庫0 | Boolean | |
| UnitPrice | 単価 | Decimal | 販売価格 |
| SalesPatternId | 販売パターンID | Int32 | 販売条件の識別 |

## CSV エクスポート手順

1. Excel で 3 シート（商品マスタ / 販売基本情報 / 販売条件）をそれぞれ CSV 形式で保存
2. 文字コード UTF-8（BOM 付き推奨）
3. CEC 管理画面 → 各インポート画面からアップロード
4. インポート順: 商品マスタ → 販売基本情報 → 販売条件

## テンプレート生成

`scripts/generate_product_reference.py` が `Products.csv` から SKU 参照表（`output/products/product_sku_reference.csv`）を生成する。  
Excel テンプレート本体は IF 設計書のサンプル列構成に合わせて手動作成し、上記関数定義を埋め込む。
