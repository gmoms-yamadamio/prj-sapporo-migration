# サイトコード / 管理グループコード（確定）

> 版: 確定  
> 更新日: 2026-07-14

## 方針

**移行元データはすべてシュパーク 1 サイトのデータ**である。  
旧サイトの `siteId` 列に `1` / `3` 等の値が混在していても、CEC への出力では **サイトコードを一律 `shupark`** とする。

**管理グループコードは、サイトコードとは別区分**であり、**成果物を問わず一律 `sp_common`** とする（**2026-07-14 確定**。従来は会員・注文系のみ `shupark` としていたが、商品系（`sp_common`）と統一）。

## 対応表

| 旧 `siteId` | CEC サイトコード | 管理グループコード | 備考 |
| --- | --- | --- | --- |
| すべて | `shupark` | `sp_common` | 旧 siteId は参照しない |

## 成果物ごとの適用

| 成果物 | サイトコード | 管理グループコード |
| --- | --- | --- |
| 注文履歴 CSV | `shupark` | `sp_common` |
| 会員ポイント CSV | `shupark` | — |
| 商品マスタ / 販売基本情報 / 販売条件 | — | `sp_common` |

## 環境ごとのシュパークサイトID（数値）

CEC管理画面上のシュパークサイトの**サイトID（数値）は環境によって異なる**。販売基本情報CSVの「掲載サイトID」列は、アップロード先の環境の値に合わせて書き換えること。

| 環境 | シュパークサイトID | 備考 |
| --- | --- | --- |
| 本番環境 | `2` | 商品系Excel（[商品系Excel.md](../migration-policy/deliverables/商品系Excel.md)）の掲載サイトIDの確定値はこちらを前提とする |
| 移行環境（検証実行用） | `1` | 検証実行で販売基本情報CSVをアップロードする場合は、掲載サイトID列の値を `2` から `1` に書き換えること |

## その他の固定値

| 用途 | 値 |
| --- | --- |
| ポイントキャンペーンコード | `migration-shupark` |

- 設定ファイル: `scripts/config/site_id_mapping.json`
